# Block P Benchmark Baseline (2026-04-11)

Scope exercised:

- `pulith-store` register hot paths (`register_artifact`, `register_extract`)
- `pulith-state` update hot paths (`state_patch` in `state_growth` bench)
- `pulith-install` commit-transition hot paths (`copy_transition` bench)

Command set used (short-run smoke profiling):

```bash
cargo bench -p pulith-store --bench register -- --sample-size 10 --warm-up-time 0.1 --measurement-time 0.2
cargo bench -p pulith-state --bench state_growth -- --sample-size 10 --warm-up-time 0.1 --measurement-time 0.2
cargo bench -p pulith-install --bench copy_transition -- --sample-size 10 --warm-up-time 0.1 --measurement-time 0.2
```

Observed hotspots to prioritize:

- `pulith-install` `copy_transition` shows repeated regressions in some adaptive threshold modes, especially larger payload ranges
- `pulith-state` `state_patch/1000` flagged measurable regression in this run

Stable/healthy signals in this run:

- `pulith-store` register paths produced consistent throughput across tested sizes
- several `copy_transition` non-adaptive and low-threshold modes improved or remained within noise

Interpretation notes:

- this pass used short criterion windows for fast iteration, so values are directional
- follow-up tuning should re-run with longer measurement windows before locking defaults

Immediate tuning focus recommendation:

1. tune adaptive threshold behavior in install transition paths
2. profile and reduce state patch overhead around medium snapshot sizes (~1000 resources)

## Follow-up tuning pass (2026-04-11, same command profile)

Code changes applied:

- `pulith-install` now stages files with an install-specific adaptive threshold of 1 MiB (`INSTALL_STAGE_COPY_ONLY_THRESHOLD_BYTES`) instead of inheriting the generic fs default
- directory staging now uses entry metadata size hints to avoid duplicate `metadata` calls per file during adaptive staging
- reliability regression guard added: replace-mode stage failure with missing stored artifact keeps prior install content unchanged

Command used:

```bash
cargo bench -p pulith-install --bench copy_transition -- --sample-size 10 --warm-up-time 0.1 --measurement-time 0.2
```

Directional observations from this pass:

- large payload ranges show measurable improvements in several adaptive and link/copy paths (notably 8 MiB and 16 MiB groups)
- 1 MiB adaptive mode still shows mixed/noisy regressions on this short window and should be rechecked with longer measurement windows before lock-in
- results remain directionally useful but variance is high at short measurement time; threshold finalization should use longer runs

Current recommendation for manager authors:

- keep install staging adaptive and bias toward earlier link-or-copy behavior for medium+ artifacts (1 MiB threshold baseline)
- treat short-run criterion regressions as triage signals; confirm with longer windows before changing manager defaults globally
