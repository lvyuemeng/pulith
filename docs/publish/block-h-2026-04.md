# Block H Publish Dry-Run Evidence (2026-04)

## Scope

Dry-run publish checks were executed for externally intended crates only.

Excluded as internal/non-publish:

- `runtime-manager-example`
- `pulith-shim-bin`
- `pulith-backend-example`

## Commands

```bash
for c in pulith-fs pulith-version pulith-resource pulith-source pulith-verify pulith-archive pulith-fetch pulith-store pulith-state pulith-install pulith-platform pulith-shim; do
  cargo publish -p $c --dry-run
done
```

## Result

All crates reported the same environment-level failure:

```text
error: crates-io is replaced with non-remote-registry source registry `ustc`;
include `--registry crates-io` to use crates.io
```

Interpretation:

- command invocation and crate scope are verified
- this environment cannot complete crates.io dry-runs because local cargo source replacement redirects `crates-io` to `ustc`
- evidence is still captured for Block H; a follow-up dry-run should be executed in a crates.io-direct environment before first real publish

## Follow-up requirement before real release

- rerun `cargo publish -p <crate> --dry-run --registry crates-io` in an environment without source replacement
- keep the crate list above as the default externally intended publish set unless policy changes
